## Ключевые решения

### 1. Поэтапный подход — подтверждён
Все участники согласились с фазированным планом. Саша: "Мне импонирует это разделение на этапность и органическое движение. Мы улучшаем то, что есть, без резких движений."

### 2. Дизайнер — внешний, на фрилансе
- Внутреннего дизайнера нет (Тимофей уволился)
- Вова: "отдельного квалифицированного специалиста в этом направлении нет"
- Алексей нашёл кандидата — человек, дизайнивший знакомую команде систему. Встреча завтра
- Процесс: оценка бюджета → утверждение с Сашей → макет → к Руслану (фронтенд)

### 3. Таск-трекер — ClickUp (пока)
- Захар предложил миграцию на Asana (PSV упёрлись в лимиты ClickUp)
- Для CAS пока ClickUp хватает
- Саша: "Буду зелёным молотком забивать гвоздь. Мне всё равно, работаем в том, что есть"
- Решение: ждём решения на уровне холдинга, пока работаем в ClickUp

### 4. Фаза 1 — работаем на текущих данных
- Никакого физического переноса из 1С не делаем
- Борис создаёт агрегированные таблицы (materialized views) из существующих данных
- Это "кэширование" — те же данные, но быстрее за счёт ClickHouse
- Для Олега: "мы не дублируем, мы просто сделали оптимально"

### 5. Self-hosted ClickHouse — запланирован переезд
- Текущее: 500 ГБ ILRD в облаке, данные пишутся, но не используются
- Вова: "на self-hosted в 4 раза больше объём, быстрее, дешевле"

### 6. Денис — свой roadmap, наша оптика
- У Дениса свои приоритеты: server bidding (аукционы с SDK на сервер), стабильность
- PSV-студии на SDK минус 2-3 версии от актуальной
- Никто не оценивает качество rollout — клиенты находят баги сами
- Боря хотел взять оценку экспериментов, но не смог (нет людей, нет инструментов)
- Решение: Алексей выровняет словари с Денисом, покажет что ILRD = его же потребности

## Новая информация для продукта

### Комиссия — toggle gross/net
Саша поднял критичный вопрос:
- CAS вычитает ~10% комиссии из данных, которые видят менеджеры
- Это занижает eCPM при сравнении с конкурентами
- Нужен переключатель: "показать с комиссией / без комиссии"
- У каждого приложения своя ставка комиссии (хранится в отдельном справочнике 1С)
- Саша: "Человек видит на уровне дашборда, он не может всё время помнить что минус 10%. Для сравнения эффективности показываем без комиссии, для выплат — с комиссией"
