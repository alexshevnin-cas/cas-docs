## 1. Payments — текущее состояние и редизайн

### Текущие проблемы (подтверждено скриншотами)

- **UI в стиле бухгалтерских проводок** — непонятен обычному пользователю
- **Adjustments путают клиентов:** сумма -$0.08 раскрывается в 5 строк по bundle'ам, занимает полэкрана
- **Два типа корректировок:**
  - Adjustment — общий страйк за фрод, распределяется на всех (мнение сети)
  - Fraud Traffic — конкретные bundle'ы с доказательным фродом
- **Множественные методы оплаты** — один клиент может иметь крипту + банк + несколько счетов (у Димы 4-5 видов выплат)
- **Комиссии** — разные fee за раннюю/обычную выплату, bank fee, crypt fee
- **Старые adjustments** за прошлые периоды приходят постоянно

### Решение (согласовано)

Проблема **только в отображении**, не в логике расчётов. Бэкенд считает корректно.

Новый UI Payments:
1. **Одна большая цифра** — текущий баланс
2. **Два варианта вывода:**
   - Снять раньше → комиссия X%
   - Снять как обычно → без комиссии
3. **Предупреждение:** "Баланс может измениться (~1.5%) из-за корректировок от сетей"
4. **История изменений:** "Было X, изменение на Y%, было Z, изменение на W%"
5. **Кнопка "Подробнее"** → drill-down в детали adjustments (для тех, кому надо)
6. **Кнопка "Связаться с менеджером"** → для вопросов по корректировкам

> Цитата: "Оказывается всё проще чем я думал. Просто чуть-чуть более грамотно переставить, чтобы людей это не смущало"

### Референсы
- Посмотреть как payments сделано у AppLovin и других игроков
- Текущее UI = "СНГовское отображение, как делали бухгалтеры из украинской фирмы"

## 5. Бизнес-инсайты

### Почему клиенты остаются (на примере Димы)
- Скидка 8% (вместо стандартных 10%)
- Кастомные выплаты (5 направлений, вручную)
- Личный менеджер (Влад Горик)
- Команда из 3 человек на него работает
- > "За эти 8% он получает мощный сервис + инсайды + дружеские отношения"

### Publishing ICP — ограничения
- **Не берут на паблишинг:** приложения (не игры) — VPN, календари, трекеры
- Причина: нет экспертизы в закупке для non-gaming
- **Зона роста:** научиться паблишить новые жанры (Sandbox/Roblox-like уже умеют)
- Клиент на $50K ушёл из-за проваленного паблишинга, сейчас возвращают

### Саппорт как конкурентное преимущество
- Техподдержка = "старший брат", а не автоматизация
- SDK слишком сложный для полной автоматизации (миллионы комбинаций)
- **Идея:** база знаний → AI-бот → сначала для внутренних, потом для клиентов
- > "Наша сила в людях. Это делает наше бизнес-преимущество"

### Presentation advice (10 февраля, "Состояние CAS")
- **НЕ показывать shiny objects** (R&D, UA automation) — Олег отвлечётся
- Показать упрощённую версию: H1 cleanup, конкретные шаги
