# Промпт: LTV Calculator — одностраничный лид-магнит

## Задача

Создай одностраничный интерактивный веб-калькулятор рекламной выручки для мобильных разработчиков. Это лид-магнит для платформы CAS.AI (ad mediation). Файл: один `index.html` со встроенными CSS и JS, без внешних зависимостей кроме Google Fonts.

---

## Бренд

- **Компания:** CAS.AI — платформа ad mediation для мобильных разработчиков
- **Цвета:** основной `#6C5CE7` (фиолетовый), акцент `#00D2D3`, фон `#0F0F1A` (тёмный), карточки `#1A1A2E`
- **Шрифт:** Inter (Google Fonts)
- **Тон:** профессиональный, но дружелюбный. Без корпоративного буллшита. Обращение к разработчику как к коллеге.
- **Логотип:** текстовый `CAS.AI` в хедере, ссылка на https://cas.ai

## Флоу калькулятора (пошаговый визард)

### Step 1: Форматы рекламы + Превью экрана
- Заголовок: "What ad formats do you use?"
- **Левая часть** (на десктопе): 3 карточки-тогла с иконками:
  - **Banner** — иконка прямоугольника внизу экрана
  - **Interstitial** — иконка полноэкранного оверлея
  - **Rewarded Video** — иконка play + монетка
- Можно выбрать любую комбинацию (минимум 1)
- **Правая часть: Phone Mockup Preview**
  - CSS-рисованный силуэт мобильного телефона (скруглённый прямоугольник с нотчем, ~300×580px)
  - Внутри "экран" — схематичная игра (серый фон + placeholder-контент)
  - При включении формата на экране телефона **анимированно появляется** соответствующий ad placement:
    - **Banner ON** → внизу экрана появляется цветная полоска (320×50) с лейблом "Banner Ad" и иконкой ×. Показать что занимает ~10% экрана снизу
    - **Interstitial ON** → поверх экрана полупрозрачный оверлей + полноэкранный блок "Interstitial Ad" с кнопкой × в углу и таймером "Skip in 5s"
    - **Rewarded ON** → кнопка "Watch Ad for Reward 🎬" появляется в центре экрана (как in-game prompt). При нажатии — тот же полноэкранный оверлей но с progress bar и "Rewarded Video"
  - Если выбрано несколько форматов — показывать их **по очереди** с автопереключением каждые 2 секунды (карусель), с индикатором какой формат сейчас показан
  - На мобилке: Phone Mockup под карточками выбора, компактнее
- Кнопка "Next" активна когда выбран хотя бы 1 формат

### Step 2: Конструктор сессии + Timeline
- Заголовок: "Build a typical user session"
- Подзаголовок: "Drag to set how many ads a user sees per session"

#### Настройки (слайдеры)
- **Session duration** — слайдер 1–10 минут, дефолт 3 мин. Это задаёт длину таймлайна
- Для каждого выбранного формата — слайдер + число:
  - **Banner:** 0–20, дефолт 6, шаг 1, подсказка "Auto-refresh every N seconds" (N = session_duration / banner_count, показывать рассчитанное)
  - **Interstitial:** 0–5, дефолт 2, шаг 1, подсказка "Fullscreen ad breaks"
  - **Rewarded:** 0–5, дефолт 1, шаг 1, подсказка "User-initiated, highest eCPM"

#### Визуальный таймлайн сессии (многотрековый)
- Горизонтальная шкала на всю ширину. Ось X = время сессии (0:00 → 3:00)
- Разметка: вертикальные засечки каждые 15 секунд, лейблы каждые 30 секунд
- **3 параллельных трека** (горизонтальные дорожки, друг под другом, как в DAW/видеоредакторе):
  - **Трек "Banners"** 🟢 — лейбл слева, зелёная дорожка. Маркеры = прямоугольные блоки (имитируют баннер), равномерно распределены. Между блоками тонкая линия (баннер виден постоянно, рефрешится). Ховер: "Banner refresh #N — every 30s"
  - **Трек "Interstitials"** 🟠 — лейбл слева, оранжевая дорожка. Маркеры = высокие блоки на всю высоту трека (фуллскрин), равномерно распределены. Ховер: "Ad break #N — fullscreen interstitial". Между блоками пустое пространство (геймплей)
  - **Трек "Rewarded"** 🟣 — лейбл слева, фиолетовая дорожка. Маркеры = ромбы/блоки с иконкой ▶, расположены в середине сессии. Ховер: "Rewarded Video #N — user opt-in"
- Треки показываются **только для выбранных форматов** (если баннеры не выбраны — трека нет)
- Вертикальные пунктирные линии от оси времени проходят через все треки — легко видеть одновременные события
- Playhead-анимация (опционально): тонкая белая вертикальная линия медленно движется слева направо, показывая "проигрывание" сессии
- **Под треками**: текстовая сводка — "Banner every 30s · Ad break every 90s · 1 rewarded mid-session"
- Таймлайн **реактивно обновляется** при движении слайдеров — маркеры плавно перемещаются/добавляются/убираются
- **Phone Mockup** (маленький, справа от таймлайна на десктопе) — при ховере на маркер показывает как этот ad выглядит на экране

- Кнопка "Next"

### Step 3: Аудитория
- Заголовок: "Tell us about your audience"
- Поля:
  - **MAU** (Monthly Active Users) — числовой инпут с пресетами: 10K, 50K, 100K, 500K, 1M
  - **Sessions per user per day** — слайдер 1–10, дефолт 3, подсказка "How often users open your app daily"
- Кнопка "Next"

### Step 4: География
- Заголовок: "Where are your users?"
- Интерфейс: распределение аудитории по регионам (в сумме = 100%)
- Регионы (с дефолтными долями):
  - **Tier 1** (US, UK, CA, AU, DE, FR, JP, KR) — 20%
  - **Tier 2** (BR, MX, TR, PL, TH, VN, MY) — 30%
  - **Tier 3** (IN, ID, PH, EG, BD, остальные) — 50%
- Слайдеры, сумма автоматически нормализуется до 100%
- Показывать текущее распределение в мини пай-чарте (CSS-only)
- Кнопка "Calculate"

### Step 5: Результат
- Анимированное появление числа (count-up от 0)
- Главное число крупно: **$XX,XXX / month**
- Разбивка под ним:
  - Revenue by format (banner / inter / rewarded) — горизонтальный stacked bar
  - Revenue by tier — 3 строки с суммами
  - Per-user metrics: ARPDAU, ARPU monthly
- Блок "Want to earn more?":
  - Текст: "CAS.AI mediation typically increases ad revenue by 20-40% through smart waterfall optimization"
  - Показать "optimized" число: текущий результат × 1.3 (подсветить разницу)

## eCPM бенчмарки (зашить в JS)

Значения eCPM в долларах за 1000 показов. Это средние рыночные данные 2025-2026:

```javascript
const ECPM = {
  banner: { tier1: 1.20, tier2: 0.45, tier3: 0.15 },
  interstitial: { tier1: 12.00, tier2: 4.50, tier3: 1.50 },
  rewarded: { tier1: 18.00, tier2: 7.00, tier3: 2.50 }
};
```

---

## Формула

```
Monthly Revenue = MAU × avg_sessions_per_day × 30 × Σ(format_impressions × Σ(geo_share × eCPM_format_geo)) / 1000
```

Пошагово:
1. `daily_impressions_per_user[format] = sessions_per_day × impressions_per_session[format]`
2. `blended_ecpm[format] = tier1_share × eCPM[format].tier1 + tier2_share × eCPM[format].tier2 + tier3_share × eCPM[format].tier3`
3. `monthly_revenue[format] = MAU × daily_impressions_per_user[format] × 30 × blended_ecpm[format] / 1000`
4. `total = Σ monthly_revenue[format]`

## Дизайн и UX требования

- **Mobile-first**, responsive. На мобилке шаги идут вертикально
- **Dark theme** — фон `#0F0F1A`, текст белый/серый
- **Анимации:** плавные переходы между шагами (CSS transitions), count-up на финальном числе
- **Progress bar** сверху — показывает текущий шаг (1/5)
- **Можно вернуться назад** на любой шаг и пересчитать
- Слайдеры стилизованы под бренд (фиолетовый трек)
- Карточки форматов с hover-эффектом и чёткой визуальной обратной связью выбора
- Финальное число — крупный шрифт, glow-эффект фиолетовым
- Без скролла в рамках одного шага (на десктопе). Всё умещается в viewport
- **Нет внешних зависимостей** — чистый HTML/CSS/JS, только Google Fonts (Inter)
- Минимализм — никаких лишних декораций, фокус на контенте
