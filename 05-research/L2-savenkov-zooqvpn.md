## Профиль
- Продукт: ZooqVPN — кросс-платформенный VPN (мобайл, десктоп, extension)
- Другие продукты: ТВ-пульт, Screen Mirroring (утилиты)
- Масштаб: несколько приложений, не игровая вертикаль
- Платформы: Android, Windows, browser extension
- Текущий стек аналитики: **AppsFlyer** (атрибуция, кастомные ивенты), **Amplitude** (продуктовая аналитика), **Firebase**
- Tenjin: НЕ используют (не обязателен для медиации)
- Опыт с CAS: ~1.5 месяца активной работы с дашбордом
- Монетизация: реклама (banner, interstitial, rewarded) для free-пользователей
- Роль в монетизации: сам балансирует рекламную нагрузку + CAS аккаунт-менеджер

## Ключевые боли

1. **Данные разбросаны по трём источникам** — Ad Spend из Facebook/Google, Ad Revenue из CAS, Events из AppsFlyer. Собирает вручную в таблицу. _"Изначально, если брать самую боль, то мне хотелось бы в одном репорте видеть мой spend по ad source, мой ad revenue... и все это разбито по GEO и по AdType."_

2. **Нет когортного LTV в CAS** — считает LTV вручную: берёт кастомные ивенты из AppsFlyer (Ad Impression Banner/Interstitial/Rewarded), умножает на eCPM из CAS, получает approximate revenue per cohort. _"Мы просто в таблицу выгружаем эти данные, и потом их умножаем на eCPM по баннеру, по Interstitial, по Rewarded."_

3. **eCPM-виджет врёт** — средний eCPM по всем ad types бесполезен (Interstitial $4 vs Banner $0.20 → average <$1). _"Этот ECPM, он очень обманчивый, потому что он берет среднее по всем add-форматам, и он, по сути, вообще тебе ничего не говорит."_

4. **UX кабинета устаревший** — попапы без отдельных URL, скрытый экспорт, невидимые пресеты, нечитаемые графики. _"Выглядит как будто не было продакт-менеджера... а был только девелопер, которому сказали — сделай."_

5. **Churn из-за рекламы — не измерить** — хочет понять, какие пользователи уходят именно из-за рекламы, но нет инструмента. _"Мне было бы интересно понимать, какие пользователи уходят... именно из-за рекламы."_

6. **Нет DAU в CAS** — получает DAU от аккаунт-менеджера (Сергея), не может сам посмотреть и сравнить с Firebase/Amplitude. _"Было бы удобно, если бы я мог видеть эти DAU здесь прям в дашборде."_

## Инсайты для продукта

### Что подтвердилось
- **App Version split — критично для L2** — подтверждает US-L2-02/03/13. _"Если бы можно было сравнивать по версии приложения Impressions и CPM, это было бы очень круто."_
- **Сравнение периодов — must-have** — inline как в Stripe, Google Play, App Store Connect. Подтверждает US-L1-02.
- **Share link — реальный use case** — скидывает фаундеру, нужно как в Amplitude. Подтверждает US-L2-15.
- **Impressions per session — актуально** — VPN специфика: балансировка рекламной нагрузки пропорционально длине сессии. Подтверждает US-L2-06.
- **DAU cross-verification нужен** — сравнение CAS SDK DAU с Firebase/Amplitude для детекции проблем SDK. Подтверждает US-L2-10.
- **Presets нужны и должны работать** — подтверждает US-L2-14, текущая реализация ненадёжна.
- **Quick View с виджетами — правильный подход** — подтверждает стратегию Инициативы 1.

### Что нового
- **Unified report (Spend + Revenue + Events)** — самая главная боль L2. Сводный отчёт из нескольких источников в одном месте. Это ключевой дифференциатор.
- **CAS ↔ AppsFlyer интеграция** — конкретный запрос на partner integration. AppoDeal уже делает подобное. Может устранить необходимость передавать impression events через приложение.
- **eCPM по ad type — не агрегированный** — средний eCPM вводит в заблуждение. Нужно показывать отдельно per ad type или убрать.
- **Metric education** — не все клиенты эксперты. Рекомендуемые метрики по дефолту (как AppsFlyer), обучение через интерфейс.
- **Кастомизируемые country tiers** — у VPN-продукта tier 1 может отличаться от игрового.
- **Desktop/Extension SDK** — CAS не монетизирует desktop/extension. Desktop-трафик дороже мобильного. Потенциальная новая ниша.
- **Ad churn analysis** — хочет понять, кто уходит из-за рекламы. Решение: A/B-тест рекламной нагрузки + когортный анализ churn.
- **Amplitude как UX-референс** — клиент привык к Amplitude, ссылается на Share link, группировку по языку, таблицу под графиком.
- **Stripe как UX-референс для Payments** — inline compare в Stripe payments.

### Что противоречит текущим планам
- **Network analysis (US-L2-04, US-L2-05) — НЕ интересен этому L2** — _"Нет нет нет, нету времени на такое"_ по поводу сетей и SDK-версий. Подтверждает, что не все L2 одинаковы — VPN-вертикаль отличается от игровой.
- **Бенчмарки нужны и для L2, не только PubC** — в стратегии бенчмарки запланированы для PubC (Инициатива 3), но L2-клиенты тоже хотят.
