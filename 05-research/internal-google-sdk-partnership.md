# Internal — Google Play SDK Partnership

**Дата:** 2026-02-12
**Респонденты:** Олег Шлямович (CEO), Александр Осыка (COO), Владимир Капацын (CTO), Денис Ещенко (SDK Lead), Алексей Шевнин (Product Owner)
**Тип:** Internal
**Контекст:** Google Play (контакт — Елена, SDK Partnerships) вышла на CAS с предложением войти в программу SDK Product Portfolio. CAS SDK используется в ~4-5K приложений на Google Play, что привлекло внимание Google. Экстренный созвон для обсуждения.

---

## Что предлагает Google

### SDK Product Portfolio
- **SDK Console** — управление версиями, статистика adoption, installs, market share, crashes
- **Inline Install API** — установка SDK-компонентов без обновления приложения
- **App Integrity API** — проверка подлинности сигнала (не эмулятор, не бот), фильтрация fraud traffic
- **Family Ads program** — легальный вход в детские приложения
- **Play Billing API** — доступ к биллингу

### SDK Policy Mitigation
- **Proactive policy violation support** — Google предупреждает до бана, а не банит молча
- **1:1 policy consultation** — персональные консультации по политикам

### SDK Console (подробно из deck)
- **Crashes & ANRs** — автоматическая атрибуция крэшей к SDK, агрегация, фильтрация по версии/устройству/Android version
- **Communication loop** — разработчики приложений шарят крэши → SDK-провайдер отвечает гайдами
- **Version management** — пометка версий как outdated, заметки для разработчиков, блокировка релизов с проблемными версиями
- **Android Studio integration** — предупреждения в IDE при использовании устаревших версий SDK
- **Deprecation notifications** — Google Play Console уведомляет разработчиков о необходимости обновить SDK

## Проблема: детские приложения

- Сейчас CAS SDK в детских приложениях **скрывает своё имя** — переименован, чтобы пройти проверку Google Play
- **Это нарушение** — Google может обнаружить и забанить
- При сертификации в SDK Product Portfolio Google увидит реальное SDK → детские приложения словят бан
- **Масштаб:** ~5 клиентов, ~$60K оборот, ~$5K дохода, 10-15% бизнеса
- **Предложение Осыки:** перевести детские приложения на AdMob/IronSource, не рисковать отношениями с Google ради 5K дохода

## Техническая сторона

### Проблема веток SDK (Денис)
- Для сертификации может понадобиться отдельная ветка SDK (CAS Kids)
- Денис: «Даже две ветки сложно. Каждый скрипт подписывается именем, тысячи файлов»
- Новому SDK нужен отдельный домен сайта, maven repository
- Google проверяет количество приложений в маркете — новое имя = 0 приложений → отклонят
- **Предложение Ещенко:** сертифицировать CAS Exchange (только для взрослых), если пропустят — хорошо, если нет — делать отдельную ветку

### Процесс сертификации (из deck)
1. Создать аккаунт в SDK Console
2. Добавить SDK по maven_id
3. Скачать verification file, встроить в SDK binary
4. Опубликовать версию с verification file
5. Ожидание верификации — до 14 дней

### Ресурсы
- Денис не может делегировать — команда ещё учится, ядро только у него
- Возможно использование AI (Claude Code) для автоматизации рутинных операций ребилда
- Олег предложил закупить платные AI-инструменты для Дениса
