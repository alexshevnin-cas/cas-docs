## Контекст в двух словах

Мы делаем BI-кабинет для наших клиентов (разработчиков мобильных приложений). Не только внутреннюю аналитику, но и немного  продукт — часть кабинета CAS.

**Подход:** сначала наполнение и логика, потом натягиваем на покупной Vue-шаблон, живём с этим в проде, обкатываем, и только потом (если надо) приходит дизайнер и полирует.

**Шаблон:** [Rixzo by Spruko](https://projects.spruko.com/vue/rixzo/preview/dashboards/sales) — Vue, готовые UX-компоненты (карточки, таблицы, графики, sidebar, фильтры).

**Прототип наполнения:** [onebi на GitHub Pages](https://alexshevnin-cas.github.io/onebi/) — React-прототип на фейковых данных..

## Итеративный план

Каждый шаг — демонстрируемый результат. Не ждём пока всё будет готово.

| Шаг | Что делаем | Результат | Данные |
|-----|-----------|-----------|--------|
| **Step 1: Каркас + Quick View** | Sidebar, навигация, 5 карточек, trend chart, network breakdown | Можно открыть, покликать, показать команде | Хардкод / JSON-файл |
| **Step 2: Reports** | Chip-фильтры, таблица, chart toggle, CSV export | Полноценный рабочий BI на моках | Мок API (тот же JSON) |
| **Step 3: Живые данные** | Подключаем реальный API вместо моков | Бета для клиентов | API бэкенда |

**Почему так:**
- Step 1 можно показать быстро — это в основном компоненты из шаблона
- Step 2 — основная сложность (таблица с фильтрами), но UI уже стоит
- Step 3 — только замена источника данных, UI не меняется. Фронт и бэк работают параллельно с шага 1


## Что делаем

### Результат первой итерации

Новый кабинет CAS целиком — не модуль внутри старого, а отдельное SPA с собственным sidebar, навигацией и авторизацией. BI — это главный (и пока единственный значимый) раздел этого кабинета.

**Структура кабинета:**

```
┌──Sidebar──┐  ┌──────────────────────────────────┐
│ CAS.AI    │  │                                    │
│           │  │  [Quick View]  [Reports]            │
│ ◉ Analytics│ │                                    │
│ ○ Apps    │  │  ← содержимое активной вкладки →   │
│ ○ Docs    │  │                                    │
│ ○ Support │  │                                    │
│           │  │                                    │
│ ──────    │  │                                    │
│ 👤 User   │  │                                    │
└───────────┘  └────────────────────────────────────┘
```

- **Sidebar** — сворачиваемое левое меню (логотип, навигация, профиль пользователя)
- **Analytics** — активный раздел с вкладками Quick View и Reports
- Apps, Docs, Support — заглушки на первое время

Первая аудитория — L1 клиенты (базовая медиация, ~$5K/мес revenue). Выкатываем как бету на отдельный URL, собираем фидбек.

### 5 карточек метрик

| Метрика | Что показывает | Динамика |
|---------|---------------|----------|
| DAU | Средний DAU за период | vs предыдущий период |
| Revenue | Сумма Ad Revenue | vs предыдущий период |
| ARPDAU | Revenue / DAU | vs предыдущий период |
| eCPM | Revenue / Impressions × 1000 | vs предыдущий период |
| Fill Rate | Fills / Requests × 100% | vs предыдущий период |

### Revenue Trend (Daily)

Multi-line chart — выручка по дням за выбранный период. Если выбрано "All Apps" — отдельная линия на каждое приложение (с легендой и цветом). Если одно приложение — одна линия.

### Revenue by Network

Stacked horizontal bar + grid-раскладка всех сетей:

```
┌── Stacked bar ──────────────────────────────────┐
│ ████ AppLovin ███ AdMob ██ Unity █ ironSource ▪ │
└─────────────────────────────────────────────────┘

  AppLovin        AdMob          Unity Ads       ironSource      Meta AN
  $5,580 (30%)    $5,232 (28%)   $3,488 (19%)    $3,140 (17%)    $2,410 (13%)
  $3.60 eCPM      $2.90 eCPM     $2.82 eCPM      $3.20 eCPM      $2.86 eCPM

  ○ Bigo Ads — No data     ○ Kidoz — No data
```

Для каждой сети: revenue, доля (%), eCPM. Неактивные сети показаны серым/красным.

### Интерактивность

- Period selector: Today / Last 7d / Last 30d / This Month / Custom
- App selector (dropdown с поиском, мультивыбор)
- Клик на карточку → переход в Reports с этой метрикой

## Экран 2: Reports

**Задача экрана:** гибкий анализ данных с фильтрами и разбивками.

### Layout

```
┌──Sidebar──┐  ┌──────────── Main Panel ───────────────┐
│           │  │                                        │
│ DIMENSIONS│  │ Filters:  [Jan 16-23 ×] [iOS ×] [+]   │
│ ☐ Date    │  │ Split:    [Date] [Ad Type] [+]         │
│ ☐ Country │  │ Measures: [Revenue] [DAU] [eCPM] [+]   │
│ ☐ Platform│  │                                        │
│ ☐ Ad Type │  │ [Apply] [Reset]    [Table|Chart] [CSV] │
│           │  │                                        │
│ MEASURES  │  │ ┌──────────────────────────────────┐   │
│ ☐ Revenue │  │ │  Date     │ Rev   │ DAU  │ eCPM │   │
│ ☐ DAU     │  │ │  Jan 16   │ $7.2K │ 12K  │ $12  │   │
│ ☐ ARPDAU  │  │ │  Jan 17   │ $6.8K │ 11K  │ $11  │   │
│ ☐ eCPM    │  │ │  ...      │       │      │      │   │
│ ☐ Fill %  │  │ │  TOTAL    │ $48K  │ 12K  │ $12  │   │
│ ☐ Impr    │  │ └──────────────────────────────────┘   │
│           │  │                                        │
└───────────┘  └────────────────────────────────────────┘
```

### Что есть в Reports

- **Фильтры:** Period (обязательный), App (обязательный), Platform, Country, Ad Type — chip-based, с × для удаления
- **Разбивки (Split):** Date, App, Platform, Ad Type, Country — порядок chips = порядок группировки, drag & drop
- **Метрики:** Revenue, DAU, ARPDAU, eCPM, Fill Rate, Impressions — выбираются из sidebar или как chips
- **Визуализация:** Table (основной) / Chart (bar или line) — toggle переключатель
- **Export:** CSV — кнопка, хорошо видимая
- **Таблица:** sticky header, sticky footer с итогами, сортировка по клику

## Вопросы для обсуждения

### 1. Шаблон и компоненты

- **Лицензия Rixzo** — можно ли в коммерческом проде? Этот шаблон или искать другой?
- **Что берём из коробки, что дописываем?** Конкретно:
  - KPI-карточки, sidebar, date picker, dropdown с поиском — скорее всего есть
  - Chip-based фильтры с drag & drop — скорее всего нет
  - Таблица со sticky header/footer, сортировкой — хватит встроенной или нужен AG Grid / TanStack Table?
  - Графики (stacked bar, multi-line, pie) — какая библиотека в Rixzo, хватит ли?
- **Dark theme** — в Rixzo есть и light и dark, насколько просто переключить?

### 2. Кабинет и навигация

Это новый кабинет целиком. В прототипе уже есть сворачиваемый sidebar с 4 пунктами (Analytics, Apps, Docs, Support) и профиль пользователя.

- **Auth:** существующая сессия CAS или новый auth?
- **URL:** `cabinet.cas.ai/analytics/quickview`, `.../reports`?
- **Sidebar:** из Rixzo или свой?
- **Apps / Docs / Support:** заглушки или редиректы?

### 3. Сроки

- Когда может взять в работу?
- Сколько на первую версию с моковыми данными?

## Чего НЕ будет в первой версии

Это важно зафиксировать, чтобы scope не поплыл:

| Фича | Когда |
|------|-------|
| Разбивка по SDK / App Version | L2 |
| Retention, LTV, когорты | L2 (требуют ILRD) |
| UA метрики (CPI, ROAS, Installs) | Pub |
| Saved Views (сохранённые пресеты) | L2 |
| Share Link / Deep Linking | L2 |
| Export Excel (только CSV) | L2 |
| Anomaly Detection | L2 |
| Мобильная версия | Не в приоритете |
| Кастомный дизайн / брендинг | После обкатки |
