## Профиль клиента
- Один из старейших клиентов CAS
- Несколько аккаунтов, портфолио из 15+ приложений
- Разные жанры: гонки, песочницы, шутеры
- Платформы: Google Play, App Store
- Самостоятельно занимается оптимизацией монетизации
- Покупает проекты и улучшает их монетизацию

## Текущее использование дашборда

> "Назвать это инструментом я не могу... это не более чем проверить, все ли сетки сегодня загрузили доход"

Дашборд используется только для **диагностики** — проверить, что данные поступают. Аналитика ведётся в Excel и Firebase.

---

## Ключевые боли

### 1. Доверие к данным
- Сетки периодически не передают данные (Pangle, AppOpen и др.)
- Нет способа быстро увидеть, что сетка "отвалилась"
- DAU в CAS расходится с Firebase и Google Play
- Нужна возможность верифицировать данные

### 2. Фильтры и группировки
- Сохранённые фильтры исчезают
- Лимит 15 приложений в группе — недостаточно
- Фильтры не сбрасываются корректно
- Нет группировки приложений в когорты

### 3. Payments/Adjustments
- Аджастменты дублируются
- Некорректная разбивка внутри (сумма не совпадает с деталями)
- Приходят старые аджастменты за прошлые периоды
- Невозможно свести финансы с бухгалтерией

### 4. UX
- Нет тёмной темы (критично!)
- Медленная загрузка
- Иконки приложений не подтягиваются

## Требуемые метрики

### Критичные (сейчас нет или работают плохо)

| Метрика | Разбивки | Зачем |
|---------|----------|-------|
| **Fill Rate** | По сетям, по форматам | Понять, где теряется выручка |
| **Revenue** | По сетям, по дням, по форматам | Диагностика "отвалов" |
| **eCPM** | По странам, по сетям | Сравнение эффективности |
| **ARPU** | По странам | Оценка изменений после оптимизации |
| **DAU** | Корректный расчёт | Сейчас расходится с Firebase |

### Желательные

| Метрика | Разбивки | Зачем |
|---------|----------|-------|
| Impressions | По сетям, по форматам, по дням | Видеть аномалии |
| Revenue по версии SDK | По версиям CAS SDK | Оценить влияние обновления SDK |
| Revenue по версии приложения | По версиям app | Найти проблемы в конкретных билдах |

### Future (UA интеграция)

| Метрика | Зачем |
|---------|-------|
| Spend | Видеть расходы на закупку |
| ROI / ROAS | Окупаемость в одном месте |
| P&L | Доходы минус расходы |

## Требуемые возможности дашборда

### 1. Визуализация аномалий
- Графики с подсветкой аномалий (красным)
- Один длинный график = одна сетка
- День за днём видеть динамику

```
Пример: если сетка 3 дня не передавала данные — подсветить красным
```

### 2. Гибкие фильтры
- Сохранение фильтров (работающее!)
- Группировка приложений в когорты (>15 apps)
- Фильтр по версии SDK
- Фильтр по версии приложения
- Корректный сброс фильтров

### 3. Разбивки (breakdowns)
- По сетям (network)
- По форматам (ad type): Banner, Interstitial, Rewarded, AppOpen
- По странам (country)
- По дням (date)
- По приложениям (app)
- По платформам (iOS/Android)
- **По версии SDK** — новое!
- **По версии приложения** — новое!

### 4. Reconciliation
- Сравнение с Firebase
- Сравнение с данными сетей напрямую
- Проверка корректности DAU

### 5. Payments
- Корректные аджастменты без дублей
- Разбивка по сетям и приложениям
- Понятная история изменений

### 6. UX
- **Тёмная тема** — обязательно!
- Быстрая загрузка
- Иконки приложений из маркетов
- Мобильная версия — не нужна

## Новые user stories из анализа

### На основе требований фронтенда

**US-L2-14: Сохранять пресеты фильтров**
> Как продвинутый клиент,
> я хочу сохранять комбинации фильтров,
> чтобы быстро переключаться между типичными отчётами.

**US-L2-15: Делиться ссылкой на отчёт**
> Как продвинутый клиент,
> я хочу скопировать ссылку на текущий вид,
> чтобы поделиться с коллегой или вернуться позже.

**US-L2-16: Экспортировать в Excel**
> Как продвинутый клиент,
> я хочу экспортировать таблицу в Excel с форматированием,
> чтобы делать дополнительный анализ локально.

**US-L2-17: Видеть актуальность данных**
> Как продвинутый клиент,
> я хочу видеть, когда данные были обновлены,
> чтобы доверять цифрам в отчётах.
