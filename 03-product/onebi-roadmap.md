# One BI — План проекта

Новый кабинет CAS с BI-аналитикой как главным разделом. Отдельное SPA, тёмная тема, Vue 3.

Этот документ — для синхронизации Alexei, Borys и Ruslan. Борис видит, когда включается. Руслан видит, что строить. Алексей видит целое.

### Главное изменение

Сейчас на b2b.cas.ai всё на одном экране: карточки, таблица, фильтры, графики — всё вперемешку. В новом кабинете разделяем на два экрана:

- **Quick View** — "всё ли ок?". Карточки с ключевыми числами, тренд, breakdown по сетям. Открыл — за 5 секунд понял ситуацию.
- **Reports** — "почему так?". Полная таблица с фильтрами, сплитами, сортировкой, экспортом. Для глубокого анализа.

## Этап 1: Каркас + Quick View (подробно)

Руслан строит скелет нового кабинета. Данные берёт из текущего бэкенда b2b.cas.ai (API рекламных сетей). Борис на этом этапе не нужен.

### Шаги

| # | Шаг | Результат |
|---|-----|----------|
| 1.1 | Выбрать шаблон (Vue 3, Tailwind, тёмная тема) | Шаблон согласован с Алексеем |
| 1.2 | Собрать каркас: sidebar, роутинг, авторизация | Можно залогиниться, видишь пустой кабинет |
| 1.3 | Sidebar: заглушки разделов (Apps, Docs, Support) | Навигация на месте, разделы = пустые страницы |
| 1.4 | Quick View: селектор приложений | Выбираешь app — данные фильтруются |
| 1.5 | Quick View: селектор дат | Пресеты 7d / 30d / 90d + custom range |
| 1.6 | Quick View: Metric Cards | 5 карточек: DAU, Revenue, ARPDAU, eCPM, Impr/DAU. Число + динамика ("▲ +8.5%") |
| 1.7 | Quick View: Revenue Trend chart | Line chart за выбранный период |
| 1.8 | Quick View: Revenue by Network | Breakdown: pie/bar по рекламным сетям |
| 1.9 | Деплой на стейдж | Живой URL (cas.ai субдомен), можно показать команде |

### Данные

Источник: текущий бэкенд b2b.cas.ai (то, что клиенты видят сейчас).

**Метрики (5 карточек):**

| Карточка | Что показывает | Откуда |
|----------|---------------|--------|
| DAU | Уникальные пользователи в день | B2B API |
| Revenue | Доход от рекламы за период | B2B API |
| ARPDAU | Revenue / DAU | Расчётная |
| eCPM | Revenue / Impressions × 1000 | B2B API |
| Impr / DAU | Impressions / DAU | Расчётная |

**Фильтры:**

| Фильтр | Тип | Значения |
|--------|-----|----------|
| App | Dropdown | Список приложений аккаунта |
| Period | Preset + custom | 7d, 30d, 90d, custom range |

**Разбивка (только для breakdown-виджета):**

| Split | Виджет |
|-------|--------|
| Network | Revenue by Network (pie/bar) |

## Этап 3: Живые данные (Борис подключается)

Борис готовит materialized views в ClickHouse (инициатива D1). Руслан переключает API. UI не меняется.

| Было | Стало |
|------|-------|
| B2B бэкенд | ClickHouse (быстрые аналитические запросы) |
| Скорость зависит от объёма данных | Стабильно 2-5 сек на любой период |
| Ограниченные периоды | Любые периоды, включая 180d, 360d |

**Что нужно от Бориса:** materialized views с теми же метриками (Revenue, Impressions, DAU, eCPM, Fill Rate, CTR, Requests, Clicks) в тех же разрезах (App, Country, Network, Ad Type, Platform, Date).

**Ограничения:** лимит 50K строк в ответе (UI показывает "уточните фильтры"), target latency < 2 сек.

**Результат:** бета для клиентов.

## Этап 4: ILRD → новые метрики + real-time

Борис подключает ILRD-данные (инициатива D3). Metrics Layer расширяется.

**Новые метрики:** Active Users, Session Length, Sessions per User, Impressions per Session, Revenue by SDK Version, Revenue by App Version.

**Новые фильтры/сплиты:** SDK Version, App Version.

**Real-time:** данные за текущие сутки (закрывает gap ~1.5 суток).

Подробности: [One BI — Glossary](onebi-glossary.md), секция "ILRD".

## Ключевые решения

| # | Решение | Почему |
|---|---------|--------|
| 1 | Отдельное SPA, не модуль старого кабинета | Своя навигация, свой URL, чистый старт |
| 2 | Шаблон → обкатка → дизайнер | Дизайнер приходит после, не до |
| 3 | Vue 3 + Tailwind + тёмная тема | Миграция с Vue 2, закрытие тех. долга. Тёмную тему просят все клиенты |
| 4 | Стартуем на B2B базе | Руслан не ждёт Бориса, строит UI сразу |
| 5 | Единый UI для B2B и Internal | Один интерфейс, разные права доступа |
| 6 | Мобилка на паузу | Desktop first, держим в уме |
