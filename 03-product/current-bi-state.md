# Текущее состояние BI в кабинете

## Назначение
Зафиксировать текущую реализацию BI-функциональности в пользовательском кабинете.

Документ описывает, что доступно клиентам сейчас, а не целевое состояние.

## Источник данных
Данные поступают из рекламных сетей (не ILRD).

Это означает:
- Только агрегированные данные от сетей
- Нет данных на уровне отдельных показов
- Нет возможности строить когортный анализ
- Точность зависит от отчётности сетей

## Доступные метрики

| Метрика | Описание |
|---------|----------|
| Impressions | Количество рекламных показов |
| eCPM | Доход на 1000 показов |
| Est. Earnings | Расчётная выручка |
| DAU | Дневная активная аудитория |
| ARPDAU | Доход на одного активного пользователя в день |
| IMP/DAU | Количество показов на одного DAU |

## Фильтры

| Фильтр | Описание | По умолчанию |
|--------|----------|--------------|
| Период | Выбор временного диапазона | 7 дней / 30 дней |
| Приложения | Выбор одного или всех приложений разработчика | Все приложения |
| Geo | Страна/регион источника трафика | — |
| Сравнение | Сравнить с предыдущим периодом (галочка) | Выкл |

## Разбивки (столбцы таблицы)

| Столбец | Описание |
|---------|----------|
| App | Приложение |
| Country | Страна |
| Date | Дата |
| Network | Рекламная сеть |
| Ad Type | Тип рекламы (Banner, Interstitial, Rewarded и др.) |
| Platform | Платформа (iOS / Android) |

Количество отображаемых столбцов можно настраивать.

## Визуализация

- Табличный вид (основной)
- Графики (альтернативный)

## Ограничения текущей реализации

- Нет экспорта данных
- Нет алертов и уведомлений
- Нет сохранённых отчётов / пресетов
