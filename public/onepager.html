Продуктовые инициативы и следующие шаги

B
OneBI — Единая платформа
Портал
В работе
Флагманский проект — каждый компонент, который мы строим здесь, модернизирует всю продуктовую экосистему. Каждая фаза расширяет аудиторию и открывает новые сценарии.
MVP на базе B2B-кабинета
ClickHouse под капотом. Новый UI: Quick View + Reports — основа для всех дашбордов. Тёмная тема.
SuperAdmin View
Все приложения сгруппированы по менеджерам, обзор портфеля, детекция аномалий.
ILRD-данные
Доход на уровне показа — сплиты по SDK/App версии, диагностика сетей, когорты. Открывает L2.
Сплитовалка
A/B-тестирование для RND. Реестр экспериментов с результатами в BI.
MMP-данные
AppsFlyer, Adjust, Tenjin — единый отчёт для UA-команды.
Дашборды паблишинга
Прибыль, ROAS, индикатор здоровья, креативная аналитика для продюсеров.
Следующие шаги
  • Запуск MVP на живых данных (зависит от D1) Руслан
  • SuperAdmin — расширить MVP админскими вьюхами Руслан + Продукт
D
Data-платформа
Данные
В работе
Инициативы дата-команды. Продукт наблюдает — это enablers и зависимости для всех продуктовых треков. Список неполный — уточнить у Бориса.
  • Материализованные представления из Network Reports
  • Миграция ClickHouse на свой сервер (500 ГБ ILRD)
  • ILRD Processing — доход на уровне показа
  • CAS Event Server — сессии, IAP, атрибуция
  • Bitrix Connector — лиды, стадии, активность AM
  • Импорт маркетинговых расходов
  • Мост атрибуции Лид→Клиент→Доход
  • Миграция данных из 1С в ClickHouse
Следующие шаги
  • Завершить мат. представления — блокирует MVP BI Борис
  • Начать ILRD-обработку после мат. представлений Борис + дата-инженер
  • Bitrix-коннектор для маркетинговой воронки TBD
S
CAS SDK
Ядро
Продакшен
Основной SDK медиации. 5000+ приложений, 1B показов/мес. 53 задачи в бэклоге Trello, 16 багов. Дедлайнов нет.
  • Инструмент управления монетизацией
  • Автоматизация релизного процесса SDK
  • Стабильность SDK — 16 багов в бэклоге
  • Новые рекламные сети — Amazon, Moloko, TaurusX
  • Server Bidding Stage 1 — 8 сетей
  • Publishing SDK — авто-инициализация, ILRD
  • Server Bidding Stage 2 — новые интеграции
  • Tracking & Events — троттлинг сессий, IAP/Attribution API
  • Cross Promo — обновить адаптер под CAS 4
  • Google MCM статус + SDK Partnership
Следующие шаги
  • Завершить SB Stage 1 (8 сетей) к апрелю Денис
  • Выпустить Publishing SDK Юра
  • Запустить Cross Promo (приоритет борды) Денис
G
Growth Marketing
Маркетинг
Планируется
Сайт cas.ai (8 языков), контент-машина, воронка PubC, реферальная программа. Цель: маркетинговые расходы 30% от прибыли.
  • Лендинг PubC «Добавь свою игру»
  • Лендинги по типу клиента (L1/L2/Pub)
  • Контент-машина — блог, Telegram, LinkedIn
  • Программа удержания PubC
Следующие шаги
  • Запуск лендингов PubC и L1 Никита
  • Запуск контент-машины Никита + Ирина
  • Витрина BI после выхода MVP Никита + Продукт
I
Internal
Внутренние инструменты
Наименее изучен
Наименее изученный блок. Набор внутренних инструментов, которые используются командами ежедневно, но не были системно описаны.
  • PSV «Зелёный сайт» — внутренний портал
  • 1С — синхронизация водопадов (waterfalls)
  • Возможная миграция из 1С на новую админку по аналогии с BI
  • Компоненты автоматизации для ускорения работы танковых команд
U
B2B Кабинет пользователя
Self-service
В работе
Пользовательская часть B2B-кабинета — всё, с чем клиент взаимодействует помимо аналитики.
  • Self-service платёжка — баланс, вывод средств, история
  • Кабинет PubC — портал для кандидатов в паблишеры
  • Обращение в сервис — тикеты, чат с поддержкой
  • Документация — встроенная база знаний, гайды по SDK
