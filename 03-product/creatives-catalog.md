# Creatives Catalog — Продуктовая спецификация

> **Версия:** 0.1 (драфт)
> **Дата:** 2026-03-16
> **Статус:** Проектирование
> **Источники:** интервью Novatska (2026-03-12), запрос Zoriana Omelchuk (CMO)

---

## 1. Проблема

Рекламные креативы (видео, баннеры, playables) управляются вручную: дизайнеры сдают файлы → координатор переименовывает → заливает на рекламные сети. Нет единого каталога, нет автоматизации нейминга, нет связи с аналитикой.

**Последствия:**
- Аналитика в Superset ломается из-за неконсистентного нейминга
- Координатор тратит время на ручное переименование и заливку
- Менеджеры не видят превью креативов в контексте перформанса
- Нет единого реестра: что создано, что запущено, что работает

---

## 2. Цели

1. **Единый каталог** креативов с поиском, превью и метаданными
2. **Автоматический нейминг** по конвенции при загрузке
3. **Связка с аналитикой** — правильный нейминг → корректные данные в Superset
4. **Снижение ручного труда** координатора

---

## 3. Пользователи

| Роль | Что делает в каталоге | Текущий инструмент |
|------|----------------------|-------------------|
| **Креативщик** (8+ чел.) | Загружает готовые файлы | Папки на диске |
| **Координатор** (Hanna) | Переименовывает, заливает на сети | Ручное переименование + ручная заливка |
| **UA-менеджер** (Zoriana Omelchuk + команда) | Выбирает креативы для кампаний, смотрит перформанс | Superset + папки |
| **Менеджер игры** | Решает, какой креатив куда залить | Чат с координатором |

## 4. Нейминговая конвенция

### Правила
1. Только латиница (ASCII)
2. Разделитель: `_` (нижнее подчёркивание)
3. Без пробелов, без кириллицы
4. Формат зависит от типа креатива

### Форматы

| Тип | Префикс | Шаблон | Пример |
|-----|---------|--------|--------|
| Видео (реклама) | `V` | `V{NNN}_{код_1С}_{appname}_{WxH}_{duration}` | `V003_7448_CarCrash_1920x1080_30s` |
| Видео (стор) | `SV` | `SV{NNN}_{код_1С}_{appname}_{WxH}_{duration}` | `SV001_7448_CarCrash_1920x1080_30s` |
| Баннер | `B` | `B{NNN}_{код_1С}_{appname}_{WxH}` | `B011_7448_CarCrash_1020x500` |
| Playable | `PLAY` | `PLAY_{NNN}_{код_1С}_{appname}_{network}` | `PLAY_001_7448_CarCrash_Unity` |

### Сезонные/тематические маркеры
- Суффикс сразу после номера, без подчёркивания: `V020UE_...`
- Примеры: `UE` (seasonal), другие TBD

### Иерархия сущностей

```
Game (код 1С = 7448, "Car Crash Simulator")
 └── Creative Concept (V003 = 3-я идея)
      └── Rendition (1920x1080, 1080x1920, 30s, 15s)
           └── Deployment (Unity Ads, Meta, Google Ads)
```

**Ключевое:** Concept — единица творческой идеи. Rendition — адаптация под сеть/формат. Deployment — факт заливки на конкретную сеть.

## 5. Модель данных

### Game
| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| code_1c | string | Код из 1С (e.g. "7448") |
| name | string | Название игры |
| short_name | string | Сокращение для нейминга (e.g. "CarCrash") |
| platform | enum | Android / iOS / Both |

### Creative (Concept)
| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| game_id | FK → Game | Привязка к игре |
| type | enum | Video / StoreVideo / Banner / Playable |
| seq_number | int | Порядковый номер (003, 011, ...) |
| seasonal_tag | string? | UE, NY, etc. (nullable) |
| description | text? | Описание концепта |
| status | enum | Draft / Ready / Active / Archived |
| created_at | datetime | Дата создания |
| created_by | FK → User | Креативщик |

### Rendition
| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| creative_id | FK → Creative | Привязка к концепту |
| width | int | Ширина (px) |
| height | int | Высота (px) |
| duration_sec | int? | Длительность (для видео) |
| file_url | string | Путь к файлу |
| thumbnail_url | string? | Превью |
| generated_name | string | Автосгенерированное имя по конвенции |
| file_size_mb | float | Размер файла |

### Deployment
| Поле | Тип | Описание |
|------|-----|----------|
| id | int | PK |
| rendition_id | FK → Rendition | Какой файл залит |
| network | enum | UnityAds / Meta / GoogleAds / YASU / Mintegral / ... |
| destination | enum | Library / Campaign |
| campaign_id | string? | ID кампании (если в конкретную) |
| deployed_at | datetime | Когда залито |
| deployed_by | FK → User | Кто залил |

## 6. User Stories

### CC-01: Загрузить креатив с автонеймингом
> Как креативщик,
> я хочу загрузить файл и указать игру + тип,
> чтобы система автоматически назвала его по конвенции.

### CC-02: Видеть каталог с превью
> Как UA-менеджер,
> я хочу видеть все креативы по игре с превью (thumbnail),
> чтобы быстро выбрать нужный для кампании.

### CC-03: Найти креатив по параметрам
> Как координатор,
> я хочу найти креатив по игре, типу, номеру или дате,
> чтобы быстро найти нужный файл.

### CC-04: Указать, куда залить креатив
> Как менеджер игры,
> я хочу отметить, на какие сети залить креатив,
> чтобы координатор знал, что делать.

### CC-05: Видеть статус заливки
> Как менеджер игры,
> я хочу видеть, залит ли креатив на нужные сети,
> чтобы знать, можно ли запускать кампанию.

### CC-06: Батчевая загрузка для A/B-теста
> Как координатор,
> я хочу загрузить пачку креативов (3-5 шт.) и пометить как A/B-группу,
> чтобы они запускались одновременно.

### CC-07: Связать креатив с перформансом
> Как UA-менеджер,
> я хочу видеть метрики перформанса (Spend, Impressions, Clicks, Installs, CPI) рядом с превью креатива,
> чтобы понимать, какой креатив работает лучше.

### CC-08: Маппинг старых креативов
> Как координатор,
> я хочу привязать старые креативы (с неконсистентным неймингом) к правильным записям в каталоге,
> чтобы аналитика работала и для исторических данных.

## 7. Текущий процесс vs Целевой

### Сейчас
```
Дизайнер → файл → Hanna переименовывает → Google Drive
                                         → Hanna вручную заливает на сети
                                         → API сетей → Superset (нейминг иногда ломается)
```

### Целевой (MVP)
```
Дизайнер → загружает в Creatives Catalog (авто-нейминг)
         → Менеджер отмечает сети
         → Координатор подтверждает и заливает (пока вручную, но с правильными именами)
         → API сетей → Superset (нейминг всегда корректный)
```

### Целевой (V2 — автозаливка)
```
Дизайнер → загружает в Catalog
         → Менеджер отмечает сети + кампании
         → Система автоматически заливает через API сетей
         → Performance data возвращается в Catalog
```

## 8. MVP Scope

### Включено (V1)
- [ ] Веб-каталог креативов: список по играм, фильтры по типу/статусу
- [ ] Загрузка файлов с автоматическим неймингом по конвенции
- [ ] Превью/thumbnail для видео и баннеров
- [ ] Метаданные: игра (код 1С), тип, размер, длительность
- [ ] Поиск и фильтрация
- [ ] Статусы: Draft → Ready → Active → Archived
- [ ] Справочник игр (код 1С + short_name)

### Не включено (V2+)
- Автозаливка на рекламные сети через API
- Перформанс-данные из Superset в карточке креатива
- A/B-группы с автоматическим запуском
- Маппинг старых креативов
- Интеграция с 1С для автосинхронизации справочника игр
