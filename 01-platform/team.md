- **Контекст:** Работает автономно, формирует команду data-инженеров. Создаёт агрегированные таблицы для ускорения B2B дашбордов. Superset — только для внутреннего использования (ограничения multi-tenancy, клиентам не даём). Передаёт данные команде Владимира для клиентских дашбордов в b2b.cas.ai

### Денис Ещенко (Denis Yeshchenko)
- **Роль:** SDK Lead (подтверждено 11.03.2026)
- **Зона:** CAS SDK, server bidding, стабильность, ILRD
- **Контекст:** Владелец SDK roadmap и Feature Track (Alpha→Beta→Stable). Server bidding = аукционы с SDK на сервер. Координирует с Олена (QA) и Юрій (R&D PM). PSV-студии на 2-3 версии позади актуальной SDK

### Юрій Вітюк (Yuriy Vityuk)

- **Зона:** Финальное решение Beta→Stable, тест-пайплайн, A/B тестирование SDK-версий
- **Контекст:** Принимает решение о переводе фичи из Beta в Stable на основе данных (eCPM, ARPDAU, crash rate). Автор методологии A/B-тестирования SDK (50/50 когорты, 500 юзеров × 5 стран). Ведёт Publishing SDK

### Олена Островерха (Olena Ostroverkha)

- **Зона:** Тестирование SDK, управление релизным циклом Stable Track
- **Контекст:** Отвечает за Stable Track: Build→QA→Pilot→Ship (цикл 7-11 дней). Контролирует качество перед каждым релизом. В команде Дениса
