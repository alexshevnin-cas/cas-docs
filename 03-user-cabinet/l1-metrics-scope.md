# Метрики кабинета для L1-клиентов

## Назначение
Зафиксировать набор метрик, фильтров и разбивок для базовых клиентов медиации (L1)
в рамках редизайна кабинета.

Документ определяет границы аналитики для L1 — только агрегированные данные,
без когортных метрик и ILRD.

## Контекст
L1-клиенты:
- Небольшой или средний масштаб
- Ограниченная потребность в глубокой аналитике
- Ориентация на простоту и стабильность

Подробнее: `02-business-model/client-types.md`

## Ключевые метрики (основной дашборд)

| Метрика | ID | Назначение |
|---------|-----|------------|
| Ad Revenue | m13 | Сколько заработано на рекламе |
| DAU | m5 | Средний размер дневной аудитории |
| Ad ARPDAU | m38 | Эффективность рекламной монетизации |
| eCPM | m20 | Доход на 1000 показов |
| Impressions | m46 | Общий объём рекламных показов |
| Fill Rate | m21 | Процент заполненных рекламных запросов |

## Дополнительные метрики (опционально)

| Метрика | ID | Назначение |
|---------|-----|------------|
| Active Users | m37 | Уникальные пользователи за период |
| Impressions per User | im1 | Среднее количество показов на пользователя |
| CTR | m48 | Кликабельность рекламы |

## Фильтры

| Фильтр | Описание | Обязательный |
|--------|----------|--------------|
| Период | День / неделя / месяц / произвольный диапазон | Да |
| Приложение | Одно или несколько приложений разработчика | Да |
| Платформа | iOS / Android / All | Да |
| Страна (Geo) | Топ-страны или группы (Tier 1/2/3) | Опционально |

## Разбивки (breakdowns)

| Разбивка | Описание | Примечание |
|----------|----------|------------|
| По дате | Динамика день / неделя / месяц | Основной способ анализа трендов |
| По приложению | Сравнение приложений разработчика | Если у клиента несколько apps |
| По платформе | iOS vs Android | Базовое сравнение |
| По типу рекламы | Banner / Interstitial / Rewarded / MREC | Структура рекламного дохода |
| По стране | Топ-5 стран или Tier-группы | Упрощённая гео-аналитика |

## Исключено из L1-scope

| Категория | Примеры | Причина исключения |
|-----------|---------|-------------------|
| Когортные метрики | cs0-cs8, Retention, LTV | Требуют ILRD |
| Метрики по сетям | n1-n10 (Network Revenue, eCPM, Fill Rate) | Избыточная детализация |
| Эксперименты | m24-m30 (A/B тесты) | Не self-service для L1 |
| UA метрики | m31-m45 (CPI, ROAS, Installs) | Отдельный продукт |
| IAP детализация | i1-i12 | Вне scope базовой медиации |

## Принципы

1. **Только агрегация** — никаких данных на уровне отдельных показов или событий
2. **Понятность** — метрики должны быть очевидны без объяснений
3. **Actionable** — данные помогают понять "всё ли в порядке", а не оптимизировать вручную
4. **Минимум шума** — не показывать то, на что клиент не может повлиять
