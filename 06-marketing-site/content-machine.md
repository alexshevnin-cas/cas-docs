# Контент-машина CAS

> **Статус:** Черновик
> **Контекст:** Системное производство контента из одной темы в несколько форматов и каналов

---

## Принцип

Одна тема → один hard content → несколько soft content → распространение по owned media.

Не придумываем контент для каждого канала отдельно. Берём экспертизу, которая уже есть внутри CAS (монетизация, SDK, A/B тесты, рынок мобильных игр), упаковываем один раз глубоко, потом нарезаем.

---

## Схема

```

   │ SOFT #1 │ │ SOFT #2 │ │ SOFT #3 │

   │ post    │ │         │ │         │
   └────┬────┘ └────┬────┘ └────┬────┘
        │           │           │

                    ▼
         ┌─────────────────────┐
```

## Hard Content (якорный материал)

Один материал в месяц. Глубокий, экспертный, с данными.

| Формат | Описание | Пример |
|--------|----------|--------|
| **Гайд** | Пошаговая инструкция | "Как оптимизировать eCPM: bid floor, waterfall, ad format selection" |
| **Исследование** | Данные из сети CAS + выводы | "eCPM trends Q1 2026: что изменилось по жанрам и гео" |
| **Кейс** | Результат клиента с цифрами | "Как ZooqVPN увеличил ARPDAU на 18% за 3 месяца" |
| **Разбор** | Анализ рынка / конкурента | "Server bidding vs waterfall: данные по 5000 приложений" |
| **Мнение** | Позиционирование CAS | "Почему мы агент, а не издатель — и почему это лучше для вас" |

Публикуется на **cas.ai/blog** — это anchor URL, на который ссылаются все derivative.

---

## Soft Content (производные)

Из одного hard content нарезаем 5-8 единиц soft content:

| # | Формат | Канал | Что берём из hard content |
|---|--------|-------|--------------------------|
| 1 | **LinkedIn post** (длинный) | LinkedIn company + личные профили | Главный тезис + 1 цифра + CTA на полный материал |
| 2 | **LinkedIn carousel** | LinkedIn | 5-7 слайдов с ключевыми выводами |
| 3 | **X thread** | X (Twitter) | 5-8 твитов: тезис → данные → вывод → CTA |
| 4 | **X single post** | X | Одна цитата / цифра с картинкой |
| 5 | **TG пост (RU)** | Telegram канал | Краткая выжимка + ссылка на блог |
| 6 | **TG пост (EN)** | Telegram канал EN | То же на английском |
| 7 | **Reddit comment/post** | r/gamedev, r/admob | Полезный ответ с контекстом, без прямой рекламы |
| 8 | **Email digest** | Email-база | Тизер в ежемесячном дайджесте |

## Каналы распространения (Owned Media)

| Канал | Тип | Аудитория | Частота |
|-------|-----|-----------|---------|
| **cas.ai/blog** | Website | SEO + direct | 1 hard content/мес |
| **LinkedIn (company)** | Social | B2B, индустрия | 2-3 поста/нед |
| **LinkedIn (личные)** | Social | Персональный бренд | По желанию авторов |
| **X (@cas_ai)** | Social | Gamedev, indie | 3-5 постов/нед |
| **Telegram RU** | Messenger | Русскоязычные девелоперы | 2-3 поста/нед |
| **Telegram EN** | Messenger | Международные девелоперы | 1-2 поста/нед |
| **Reddit** | Community | r/gamedev, r/admob | Ситуативно, в контексте дискуссий |
| **Email** | Direct | Текущие + потенциальные клиенты | 1 дайджест/мес |

## Темы (из экспертизы CAS)

### Монетизация (основной блок)
- Bid floor оптимизация
- eCPM decay и как с ним бороться
- Rewarded vs Interstitial: когда что работает
- Fill rate: почему падает и что делать
- MREC как альтернатива interstitial

### SDK и техническое
- Стабильность SDK-релизов (боль клиентов)
- Server bidding: что это и зачем
- Как оценить влияние SDK-обновления на выручку

### Рынок
- Тренды eCPM по жанрам (данные CAS)
- Сезонность рекламного рынка (Q1 down, Q4 up)
- Какие жанры растут (из данных 5000+ приложений)

### Паблишинг
- Как валидировать игру за 3 дня (PubC flow)
- CPI vs Retention: что важнее при оценке
- Predictive LTV: зачем и как

### Позиционирование
- "Агент, а не издатель" — дифференциатор от Azur/DACI/Supersonic
- Прозрачность vs чёрный ящик
- Desktop SDK монетизация — новая ниша без конкурентов

## Процесс производства

```
Неделя 1:  Выбор темы → Research → Драфт hard content
Неделя 2:  Редактура → Финализация → Публикация на blog
Неделя 3:  Нарезка soft content → Планирование постов
Неделя 4:  Публикация soft content по каналам → Анализ
```

### Роли
| Роль | Кто | Что делает |
|------|-----|-----------|
| Автор темы | Эксперт CAS (Олег, Денис, BD) | Экспертиза, данные, цитаты |
| Райтер | ? (фриланс или внутренний) | Текст hard content |
| Контент-менеджер | ? | Нарезка soft content, планирование, публикация |
| Дизайнер | Тот же фрилансер? | Визуалы для carousel, OG images |

## Метрики контент-машины

| Метрика | Что измеряет |
|---------|-------------|
| Blog views | Охват hard content |
| Blog → Signup rate | Конверсия контента в регистрации |
| LinkedIn impressions | Охват soft content |
| LinkedIn engagement rate | Резонанс (likes, comments, shares) |
| X impressions + clicks | Охват + переходы |
| TG views + forwards | Охват в мессенджере |
| Email open rate + CTR | Эффективность дайджеста |
| Referral source in GA | Какой канал приводит на сайт |
