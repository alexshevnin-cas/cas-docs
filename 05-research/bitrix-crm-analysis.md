# Анализ CRM Битрикс — Pipeline и конверсия

> **Дата:** 2026-02-16
> **Источник:** Битрикс24 CRM 
> **Период данных:** июль 2024 — 16 февраля 2026
> **Всего лидов:** 3,129 (после очистки: 3,111)
> **Исключено:** Second Account (внутренние/дубли) — 16 записей, 2 нерелевантных до 2024-07

### Методология

Выгрузили все лиды из Битрикса через API. Сделки (Deals) в CRM почти не используются — 17 штук за всё время, поэтому считаем только по лидам. «Клиент» здесь — это лид, дошедший до статуса «Успешный клиент CAS». Цифры нужно перепроверить с командой — могут быть расхождения с реальным количеством активных клиентов.

«Актив%» — доля лидов, которые продвинулись дальше квалификации и не отвалились.

Важный нюанс: 178 из 184 лидов со статусом «Успешный клиент» были переведены в этот статус одним днём — 5 ноября 2025. То есть статусы проставили задним числом, а не по мере реальной конверсии. Из-за этого по данным CRM нельзя посчитать, сколько времени на самом деле проходит от заявки до клиента. Lookback 2-3 месяца в прогнозе — это допущение, а не факт из данных.

### Вывод

**Квалификация.** 88% лидов отваливаются до стадии MQL. Но после квалификации конверсия высокая: qualified → клиент = **48%** (зрелые когорты).

Этапы от чата до A/B test проходят гладко (94-98%). Основная потеря — на последнем шаге: прошёл A/B → клиент (56%).

## 6. Комментарии

### Что работает

1. **Кажется, качество обработки растёт** — Актив% с 7% до 15-16%, улучшение вдвое
2. **Объём лидов снизился, но качество не просело** — при меньшем потоке доля qualified держится на уровне 15-16%, конверсионные этапы работают так же. Однако объективно оценить качество невозможно: OPPORTUNITY не заполняется, нет оценки лидов в потенциальных деньгах → **заполнять OPPORTUNITY** хотя бы по шкале L0=$3K / L1=$6K / L2=$15K
3. **Воронка после квалификации работает** — 94-98% проходимость от чата до A/B test
4. **Зрелая конверсия qualified → client = 48%** — конверсия от всех лидов = 4-9%, от qualified = 48%
5. **Client-канал (повторные) стабилен** — конверсия 26%, не зависит от общей просадки

### Что пока непонятно

1. **Publishing pipeline почти нет** — 4% от всех лидов в CRM → **добавить поле "тип лида"** (mediation / publishing)
2. **Как привязать HubSpot** - надо обучение от Игоря и Саши

## Приложение A. Структура CRM

### Два пайплайна сделок (Deals)

| ID | Название | Использование |
|----|----------|---------------|
| 0 | Mediation | 17 сделок (не используется активно) |
| 1 | Publishing | 0 сделок |

**Основной pipeline ведётся через Лиды (Leads), не через Deals.**

### Воронка лидов (15 стадий)

```
NEW (Не обработанные)
  → IN_PROCESS (В работе)
    → Звонок BD
      → Нет ответа          ← отвал
      → Собеседование
        → NON SQL            ← отвал
        → NON MQL            ← отвал
        → MQL
          → Создан Чат
            → Интеграция
              → Интеграция 5-10%
                → A/B test
                  → Прошёл A/B test
                    → Отказавшиеся    ← отвал
                    → Upsale push
                      → Перевёл все игры на CAS
                        → Успешный клиент CAS  ← WIN
  → Закрытый (JUNK)        ← отвал
```
