# Финансовая модель CAS.AI

> **Версия:** 1.0
> **Дата:** 2026-02-16
> **Статус:** Рабочий документ
> **Принцип:** Модель — не план, а инструмент для диалога.

---

## 1. Структура выручки

```
Total Revenue = Old Users
              + New Users (Mediation)
              + New Users (BD)
              + Publishing Profit
              + RND Uplift
              + VIP Setup Uplift
              + Customer Resources Uplift
              − Force Majeure
```

---

## 2. Old Users (существующая база)

Выручка от клиентов, которые были на начало года.

### Формула

```
Old Users[m] = Old Users[m-1] × Churn Coef[m] × Seasonality[m] + Prevention[m]
```

### Параметры

| Параметр | Значение | Источник |
|----------|----------|----------|
| Old Users Base (январь) | $1,494,571 | Факт |
| Natural Churn (годовой) | 8% (к декабрю остаётся 92%) | Confirmed |
| Prevention by Accounting | +1% точечно в отдельные месяцы | Assumption |
| Seasonality Q1 (январь) | 0.9 | Confirmed |
| Seasonality Q2-Q3 | 1.0 | Confirmed |
| Seasonality Q4 (окт-дек) | 1.05–1.1 | Confirmed |

### Причины churn
- Закрытие приложения/бизнеса
- Переход к конкурентам (AppLovin MAX, ironSource)
- Недовольство сервисом

## 3. New Users — Mediation (Inbound)

Новые клиенты из входящих каналов на медиацию.

### Формула

```
New Users Mediation[m] = Pipeline DAO[m] × Conversion Rate[m]
```

**Важно:** lookback period = 2-3 месяца. Клиент, пришедший в январе, в январе денег не показывает.

### Pipeline

Источник: **Битрикс** (полный pipeline: inbound + outbound).

```
Pipeline DAO = Σ (Lead Count[channel] × Avg DAO[segment])
```

### Каналы привлечения (Inbound)

| Канал | Типичные сегменты | Годовой план (клиенты) |
|-------|-------------------|------------------------|
| Ads Channel (Google, FB) | L1, L2 | ~18 |
| TG / Reddit (органика) | L1, L2 | ~26 |
| Partnerships / Influencers | L1, L2 | ~15 |
| Organic / SEO | L1, L2 | ~11 |
| Referral (текущие клиенты) | L1, L2 | ~15 |

### Сегменты лидов (ICP)

| Сегмент | Gross Revenue/мес | Avg DAO для взвешивания |
|---------|-------------------|-------------------------|
| L0 (cold) | — | $3,000 |
| L1 (warm) | $5,000 | $6,000 |
| L2 (hot) | $12,500 | $15,000 |
| L3 | $50,000 | $50,000 |
| Unicorn | $150,000 | $150,000 |

### Конверсия

| Параметр | Значение | Статус |
|----------|----------|--------|
| Pipeline to Client | 20–25% | Целевая |
| Split: медиация / publishing | 40% / 60% | Assumption |

## 5. RND Optimal Setup Uplift

Прирост выручки от новых фич SDK и автоматизации оптимизации монетизации.

### Формула

```
RND Uplift[m] = Old Users[m] × RND Uplift %[m] × SDK Rollout %[m]
```

### Параметры

| Параметр | Начало года | Конец года | Статус |
|----------|-------------|------------|--------|
| RND Uplift % | 1% | 8% | Target |
| SDK Rollout % | 60% | 80% | Target |

### Ограничения
- Без BI невозможно точно оценить uplift "средний по больнице"
- Зависит от скорости раскатки SDK у клиентов
- Текущий эффект ~$10K/мес — пока игнорируется в модели

## 6. VIP Monetization Setup

Прирост от ручной настройки монетизации командой CAS для крупных клиентов.

### Формула

```
VIP Uplift[m] = Old Users[m] × VIP Coverage %[m] × VIP Uplift %
```

### Параметры

| Параметр | Значение | Статус |
|----------|----------|--------|
| VIP Uplift % | 15% (или 8%?) | Hypothesis, нужен анализ |
| VIP Coverage % | 25% → 35% к Q4 | Расширение планируется |
| Факт января | $48K из $60K плана (80%) | Fact |

### Открытые вопросы
- Реальный uplift: 15% или 8%? Нужен A/B или ретро-анализ
- Можно ли охватить >25% клиентов без потери качества?

## 7. Customer Free Resources Uplift

Прирост выручки приложения за счёт делегирования рутины CAS. Клиент экономит ресурсы → инвестирует в продукт → растёт выручка.

### Формула

```
Customer Uplift[m] = Old Users[m] × 10%
```

| Параметр | Значение | Статус |
|----------|----------|--------|
| Uplift % | 10% | Confirmed |

---

## 8. Publishing

Отдельное направление. Клиент на паблишинге генерирует ~5x выручки относительно медиации за счёт UA-инвестиций CAS.

### Формула

```
Publishing Revenue[m] = Active Pub Clients[m] × Avg Revenue × Profit Share

Active Pub Clients[m] = Pipeline Pub[m] × (1 − Pub Churn %) × Conversion

Publishing Profit = Publishing Revenue × Profit Share
```

### Параметры

| Параметр | Значение | Статус |
|----------|----------|--------|
| Growth Multiplier | 5x | Hypothesis (нужен ретро за 2025) |
| Profit Share (наша доля) | 50% | Confirmed |
| Pub Pipeline Churn | 30% | Assumption |
| Pub Leads (январь → декабрь) | 5 → 32/мес | Target |
| Avg DAO Pub Lead | $6,000 (L1 средний) | Assumption |

### Pipeline Publishing

```
Pub Pipeline[m] = Pub Leads[m] × Avg DAO × (1 − Churn)
Pub Revenue[m] = Pub Pipeline[m] × Growth Multiplier × Profit Share
```

### Нужна валидация
- Ретроспективный анализ Publishing за 2025: сколько пришло, сколько обработали, средний рост
- Реальный коэффициент: 3x? 5x? Зависит от жанра и GEO

## 9. Force Majeure

Потери от внешних факторов: проблемы рекламных сетей, сезонные просадки крупных клиентов.

### Компоненты

| Фактор | Макс. потери/мес | Частота |
|--------|------------------|---------|
| Турецкие музыкальные приложения (сезонные VIP) | −$180,000 | Q1, Q4 |
| Digital Turbine (проблемы сети) | −$20,000 | Нерегулярно |
| Other Networks (Pangle и др.) | Variable | Нерегулярно |

### Правило для анализа
При расчёте трендов **исключать сезонных клиентов**. Пример: без двух турецких клиентов компания в январе не просела, а выросла на $113K.

## 10. Источники данных (CRM)

### Битрикс (основной)

Полный pipeline: inbound (органика, ads, TG, Reddit, partnerships) + outbound (BD).

```
Битрикс
├── Inbound Mediation
├── Inbound Publishing
└── Outbound BD (дублируется в HubSpot)
```

**Задача:** написать скрипт выгрузки из Битрикс API для автоматического обновления pipeline.

### HubSpot (BD-срез)

Только BizDev pipeline (Игорь, Вика). Подмножество данных Битрикса.

### Целевые показатели Pipeline

| Метрика | Значение | Динамика |
|---------|----------|----------|
| Total Leads/мес | 50 → 105 | Рост |
| Total DAO Pipeline | $1.5M → $2.6M | Рост |
| Mediation Leads | 40% | — |
| Publishing Leads | 60% | — |
