# Аудит консистентности документации

Проверь, что все документы в проекте согласованы между собой.

## Проверки

### 1. User Stories ↔ Метрики
Прочитай `03-product/user-stories.md` и `04-bi/metrics-dictionary.md`.
- Каждая метрика, упомянутая в user stories, должна быть в словаре метрик
- Каждая метрика в словаре должна использоваться хотя бы в одной user story

### 2. Метрики ↔ Фильтры
Прочитай `04-bi/metrics-dictionary.md` и `04-bi/filters.md`.
- Уровни доступа (L1/L2/Pub/PubC) должны совпадать

### 3. Client Types ↔ User Stories
Прочитай `02-business-model/client-types.md` и `03-product/user-stories.md`.
- Для каждого типа клиента должны быть user stories

### 4. Design Spec ↔ Метрики
Прочитай `03-product/design-spec-bi.md` и `04-bi/metrics-dictionary.md`.
- Метрики в wireframes должны быть в словаре

### 5. Strategy ↔ Всё остальное
Прочитай `03-product/product-strategy.md`.
- Фазы и приоритеты должны соответствовать реальному состоянию документов

### 6. Битые ссылки
Проверь все ссылки вида `path/to/file.md` во всех документах — файлы должны существовать.

## Вывод

Покажи таблицу:

| Проверка | Статус | Проблемы |
|----------|--------|----------|
| ... | OK / WARN / FAIL | описание |

И список конкретных действий для исправления.
