## Результаты и кейсы

### Какие успешные кейсы есть?

Примеры проектов на паблишинге:

| Проект | Тип | Примечание |
|--------|-----|-----------|
| MultiCraft | Паблишинг | $1M+ суммарный revenue за ~1.5 года, ~$40K/мес доход разработчику |
| Sniper | Внутренний проект | Можно делать кейс-стади (внутренний IP) |
| Drive CSX (Car Crash Simulator) | Паблишинг | Активный кейс, ссылки: [Google Play](https://play.google.com/store/apps/details?id=com.honanStudio.driveX&hl=en), [App Store](https://apps.apple.com/ru/app/drivecsx-car-crash-simulator/id6736580540) |

Успешных экшенов на паблишинге пока нет. Есть внутренний проект в жанре action с объёмом закупки $350K/мес.

Средний доход с игры на паблишинге: $1–5K/мес.

### Сколько студия может заработать?

Зависит от жанра и продуктовых метрик. Типичный диапазон — $1–5K/мес на игру. Топовые проекты (MultiCraft) — ~$40K/мес для разработчика. Суммарно за жизненный цикл — от десятков тысяч до $1M+.

## Экономика

### Что входит в затраты со стороны издателя помимо рекламного спенда?

Всё включено в profit share, отдельно не тарифицируется:

- **UA** — закупщики трафика (UA managers) + креативы + ASO
- **Аналитика** — внутренние дашборды, анализ когорт
- **Продуктовое сопровождение** — продюсер, геймдизайнер
- **Атрибуция** — Tenjin (оплачивается CAS)
- **Для крупных проектов** — обсуждаемо подключение QA и арт-команды в помощь

Зарплаты команды CAS НЕ выставляются отдельно — покрываются из доли CAS в profit share.

### Какая модель разделения доходов?

**Базовая модель: 50/50 от Uplift.**

Пример:
- Игра зарабатывала **$2 000/мес** до паблишинга
- CAS вложил **$50 000** в рекламу
- Доход стал **$70 000/мес**
- Uplift = $70 000 − $2 000 − $50 000 = **$18 000**
- **CAS получает:** $50 000 (возврат рекламы) + $9 000 (50% uplift) = $59 000
- **Разработчик получает:** $9 000 (50% uplift) + $2 000 (baseline) = **$11 000**

**Альтернативная модель: 80/20 с покрытием burn rate.**
Доступна для студий с успешным трек-рекордом. CAS покрывает burn rate разработки, процент пересматривается в пользу CAS (80/20).

## Команда

### Кто будет работать над проектом?

При спенде **$200 000/мес** — выделенная команда в высоком приоритете:

| Роль | Кол-во | Что делает |
|------|--------|-----------|
| Продюсер | 1 выделенный | Стратегия, приоритизация, продуктовое сопровождение |
| Креативный дизайнер | 2 | Видео-креативы, визуал |
| UA-менеджер (закупщик) | 1 | Закупка трафика |
| Монетизатор | 1 | Настройка и оптимизация монетизации |

Если спенд меньше — команда делится в пропорции. Внимание распределяется по перспективности: некоторые проекты получают 10%, некоторые — 90%.

### Кто делает креативы? Playable?

Внутренняя креативная команда — **30–40 человек**.

- Работают с момента, когда видим возможность для скейла (мин. спенд от $100/день)
- Продолжают до момента, который считаем ликвидным
- Минимальный порог интереса: **$30–50K спенда/мес**

Playable — внутренняя разработка (уточнять у продакшн-команды).

## Процесс

### Как выглядит путь: тест → софтланч → запуск?

**Этап 1: CPI-тест (без контракта)**
- Бюджет: на **1 000 юзеров**, WW (worldwide)
- Целевые метрики:
  - R1 (Retention Day 1) > **25%**
  - Playtime per day per user: **17+ минут**

**Этап 2: Тест монетизации (без контракта)**
- Запускается, если первый тест успешен
- Целевые метрики: ROAS D7 **80–100%** (зависит от типа продукта)
- Параллельно смотрим продуктовые метрики

**Этап 3: Подписание контракта**
- Если получается тратить **>$1 000/неделю** → переходим к контракту
- Контракт подписывается после первого теста retention

**Этап 4: Скейлинг**
- Целевой ROAS: **90–130%** на **Day 120**
- Органику не считаем (бонус)

### Как выйти из издательства если всё пошло плохо?

- **Тесты проходят без договора** — можно разойтись на любом этапе тестирования
- Если видим перспективу скейла → подписываем контракт
- Контракт можно **разорвать по обоюдному согласию**
- Предупреждение о разрыве: **за 60 дней**
- При разрыве CAS получает доход от **своих когорт** (пользователей, привлечённых за время сотрудничества)

## Аналитика и A/B тесты

### К какой аналитике будет доступ?

- **Сейчас:** доступ к Tenjin (атрибуция, UA-метрики). По запросу — скриншоты из внутренних дашбордов
- **В работе:** клиентские дашборды в кабинете b2b.cas.ai (таблицы, фильтры, метрики). Внутренняя аналитика (Superset) остаётся только для команды CAS — клиентам не даём из-за ограничений изоляции данных
- **Фаза 2a:** когорты, SDK-разбивка
- **Фаза 2b:** A/B тесты в кабинете
- **Фаза 3:** Unified Report с MMP-данными

### Есть ли внутреннее решение для A/B тестов?

- **Firebase** для раздачи конфигов (Remote Config)
- Анализ результатов — внутри команды CAS через Superset, клиенту — через кабинет b2b.cas.ai (в разработке)
- Полноценный инструмент для A/B-тестов (сплитовалка + experiment registry) — в плане на фазу 2b, после ILRD

## Технологии

### Есть ли единый SDK (как Voodoo YSO)?

Нет единого SDK. Для работы по схеме паблишинга требуется интеграция **3 SDK**:

| SDK | Назначение | Примечание |
|-----|-----------|-----------|
| **CAS SDK** | Монетизация (ad mediation) | Обязательно |
| **Google Firebase SDK** | Аналитика, Remote Config, A/B | Чаще всего уже есть в приложении |
| **Tenjin SDK** | Атрибуция UA | Обязательно для трекинга |

CAS Publishing SDK (объединённый) — в планах, задача у SDK-команды (Юра).

### Почему Tenjin, а не Adjust?

1. **Самая низкая цена атрибуции** на рынке
2. **CAS — один из крупнейших клиентов Tenjin** → полный саппорт, выделенный менеджер
3. **Кастомные фичи** — визуал и функциональность сделаны по запросу CAS
4. **Удобно показывать партнёрам** — интерфейс адаптирован для PubC
5. **Автоматизации в SDK**: автоинициализация, автопрокидка IRLD
6. **Фаза 3** — возможность подключения клиентского Adjust/AppsFlyer через MMP-коннекторы

## Юридическое

### У кого IP?

IP остаётся у разработчика. Подробности — в контракте.

### На чьих аккаунтах игры?

**Крайне желательно** перенести на аккаунты CAS — но это обсуждаемо. Мягкая формулировка, не жёсткое условие.

### Можно посмотреть пример договора?

Да, пример предоставляется по запросу. Контракт подписывается **после первого успешного retention-теста**, не до.
