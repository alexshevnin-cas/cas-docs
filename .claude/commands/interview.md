# Обработка интервью

Пользователь прислал транскрипт интервью с клиентом. Обработай его в 3 шага:

## Шаг 1: Структурированный анализ

Прочитай CLAUDE.md — там шаблон для анализа интервью. Создай файл в `05-research/` по шаблону.

Имя файла: `[тип]-[имя-или-компания].md` (например `pub-gamestudio.md`).

Особое внимание:
- Выдели **конкретные боли** с цитатами
- Сопоставь запросы клиента с существующими user stories из `03-product/user-stories.md`
- Отметь NEW для запросов, которых ещё нет в user stories
- Запиши action items с указанием, какой файл нужно обновить

## Шаг 2: Обнови продуктовые документы

По action items из шага 1:
- Новые user stories → добавь в `03-product/user-stories.md`
- Новые метрики → добавь в `04-bi/metrics-dictionary.md`
- Новые фильтры → добавь в `04-bi/filters.md`
- Изменения приоритетов → обнови `03-product/product-strategy.md`
- Новые боли/инсайты → обнови `03-product/cabinet-overview.md` если нужно

При обновлении существующих файлов: добавляй, не переписывай. Пометь источник `(из интервью [имя], [дата])`.

## Шаг 3: Обнови research log

Добавь строку в таблицу `05-research/research-log.md`.

## Финал

Покажи саммари:
- Сколько новых user stories добавлено
- Сколько существующих подтверждено
- Топ-3 инсайта
- Что противоречит текущим планам (если есть)
