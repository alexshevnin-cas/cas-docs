# Онбординг в кабинете CAS

## Назначение
Описать путь нового клиента от регистрации до первого успешного опыта в кабинете.

Онбординг — критическая точка воронки: здесь клиент либо становится активным пользователем, либо уходит.

---

## Цели онбординга

### Для клиента
1. Понять, что CAS может дать
2. Интегрировать SDK и увидеть первые данные
3. Убедиться, что всё работает
4. Знать, куда обращаться за помощью

### Для CAS
1. Квалифицировать клиента (определить тип)
2. Собрать данные для персонализации
3. Создать первый успешный опыт (aha-moment)
4. Заложить основу для upsell

---

## Точки входа

| Источник | Ожидания клиента | Приоритет онбординга |
|----------|------------------|----------------------|
| Органика (сайт) | Изучает варианты | Показать ценность |
| Реферал | Уже знает о CAS | Быстрая интеграция |
| Контент (блог, гайд) | Решает конкретную задачу | Связать с задачей |
| Партнёр/конференция | Личный контакт был | Персональный подход |
| Миграция с другой медиации | Сравнивает | Показать преимущества |

```
│  ○ Более 10 приложений                                      │
│                                                             │
│  Какой примерный DAU?                                       │
│  ○ < 10K                                                    │
│  ○ 10K - 100K                                               │
│  ○ 100K - 1M                                                │

│                                                             │

│  ☐ Хочу подключить монетизацию                              │

│  ☐ Интересует паблишинг                                     │
│  ☐ Хочу улучшить текущие показатели                         │
│                                                             │
│                                        [Продолжить →]       │
└─────────────────────────────────────────────────────────────┘
```

**Логика квалификации:**

| Ответы | Предполагаемый тип | Путь онбординга |
|--------|-------------------|-----------------|
| Нет приложений + интересует паблишинг | PubC | Путь валидации |
| 1-3 apps, < 100K DAU | L1 | Базовый путь |
| 4+ apps или > 100K DAU | L2 | Расширенный путь |
| Интересует паблишинг + есть трафик | Pub candidate | Персональный контакт |

**Данные для сбора:**
- Email (обязательно)
- Название компании (опционально)
- Ссылка на приложение в сторе (опционально, но желательно)
- UTM-метки источника

## Онбординг по типам клиентов

### PubC: Путь валидации

**Фокус:** Помочь понять, работает ли прототип.

**Специфика:**
- Упрощённая регистрация (минимум полей)
- Акцент на базовые метрики (DAU, Retention)
- Ранний контакт с менеджером если есть потенциал
- Образовательный контент про product-market fit

**Дополнительные шаги:**
```
После первых данных:
┌─────────────────────────────────────────────────────────────┐
│  📈 Как понять, готов ли прототип к масштабированию?       │
│                                                             │
│  Ваши текущие показатели:           Бенчмарк категории:    │
│  • Retention D1: 28%                 • Хорошо: > 25%       │
│  • Retention D7: 12%                 • Хорошо: > 10%       │
│  • Avg Session: 4.2 мин              • Хорошо: > 3 мин     │
│                                                             │
│  ✅ Ваш прототип показывает хорошие результаты!            │
│                                                             │
│  [Узнать о паблишинге с CAS →]                             │
└─────────────────────────────────────────────────────────────┘
```

---

### L1: Базовый путь

**Фокус:** Быстрая интеграция, стабильная работа.

**Специфика:**
- Стандартный онбординг (описан выше)
- Минимум отвлечений на "продвинутые" функции
- Чёткие инструкции без вариативности
- Quick View как основной экран

**После онбординга:**
- Показать Quick View с их данными
- Объяснить основные виджеты
- Дать ссылку на FAQ

---

### L2: Расширенный путь

**Фокус:** Показать глубину аналитики.

**Специфика:**
- Предложить импорт данных из предыдущей медиации
- Показать Reports сразу после Quick View
- Рассказать про фильтры и разбивки
- Предложить настроить Saved Views

**Дополнительные шаги:**
```
После первых данных:
┌─────────────────────────────────────────────────────────────┐
│  🔧 Настройте кабинет под себя                             │
│                                                             │
│  ☐ Импортировать данные из AppLovin / ironSource           │
│  ☐ Настроить разбивку по сетям                             │
│  ☐ Создать группы приложений                               │
│  ☐ Сохранить первый пресет фильтров                        │
│                                                             │
│  [Настроить сейчас]  [Напомнить позже]                     │
└─────────────────────────────────────────────────────────────┘
```

---

### Pub: Персональный путь

**Фокус:** Белые перчатки, персональный менеджер.

**Специфика:**
- Онбординг с менеджером (созвон или чат)
- Кастомная настройка дашборда
- Обучение команды клиента
- Настройка интеграций (UA, attribution)

## Метрики онбординга

| Метрика | Описание | Цель |
|---------|----------|------|
| Registration → App Added | Конверсия в добавление приложения | > 80% |
| App Added → SDK Integrated | Конверсия в интеграцию | > 60% |
| SDK Integrated → First Data | Конверсия в первые данные | > 90% |
| Time to First Data | Время до первых данных | < 48 часов |
| Onboarding Completion Rate | Завершили весь онбординг | > 50% |
| 7-day Retention | Вернулись через 7 дней | > 40% |
